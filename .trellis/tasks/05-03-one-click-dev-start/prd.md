# PRD: 一键启动前后端

## 背景
当前开发需要分别在前端根目录和 `server/` 目录执行 `npm run dev`，启动路径割裂，成本较高。

## 目标
- 提供一个根目录一键启动入口，能够同时启动前端和后端。
- 启动后自动打开前端开发页面。
- 保留前端和后端各自独立的开发启动方式，便于单独调试。

## 方案范围
- 根目录新增一个聚合开发脚本。
- 聚合脚本并行启动前端和后端。
- 聚合脚本在前端可访问后自动打开浏览器到前端地址。
- 文档同步更新，说明新的启动方式。

## 非目标
- 不修改前后端业务逻辑。
- 不改动前后端通信协议。
- 不引入新的运行时依赖，除非实现聚合启动确实需要。

## 约束
- 前端仍保持可单独启动。
- 后端仍保持可单独启动。
- 默认应兼容当前本地开发环境，前端地址以现有 Vite 默认端口为准。
- 聚合启动应显式暴露启动失败，而不是静默忽略。

## 验收标准
- 在项目根目录执行一个命令即可同时启动前端和后端。
- 浏览器会自动打开前端开发页面。
- 现有单独启动命令仍可用。
- `README.md` 中的启动说明已更新为新流程。
